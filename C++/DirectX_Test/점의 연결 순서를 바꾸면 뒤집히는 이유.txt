점의 연결 순서를 바꿧는데 그림이 뒤집혀 보이지 않는 이유

- 가정1
그림이 보이는 면과 보이지 않는 면이 있다.
처음에 그림이 보이는 면을 화면에 출력하도록 꼭짓점을 지정 했기 때문에 그림을 볼 수 있었고,
이후 좌우 꼭짓점을 변경하자 뒷면(그림이 보이지 않는 면)이 앞으로 나와 그림을 볼 수 없게 된 것이다.

- 가정2
에초에 그림을 볼려면 원하는 그림의 형태와 같은 위치에 꼭짓점이 설정되어야 한다. 
(가정 1)과 유사 할 수 있지만 다른점은 값을 제대로 부여한다면 그림을 볼 수 있을 것이다.


실행 순서. 함수이름/ 역할
1. winMain()	/ 프로그램 메인 실행 부, 프로그램이 실행되면 가장 먼저 실행되는 함수, 아래 함수들이 차례로 호출됨
2. CreateWindow()	/ 그림을 출력하기 위해 먼저 윈도우창을 사용해야 함, 그래서 필요한 윈도우 창의 크기, 이름등을 초기화 해주기 위해 함수를 호출
3. InitD3D()	/ 다이렉트 3D를 사용하기 위해 필요한 버퍼와 값을 초기화 하는 함수
4. InitVB()		/ 출력하기 위한 그림 값을 버퍼에 저장(초기화)하기 위해 호출되는 함수
5. UpdateWindow()/ 없어도 출력이 가능하다. ( 왜 호출해야 되는지 궁금함 )
6. ShowWindow()	/ 주석을 걸면 빌드는 가능하지만 화면에 윈도우 창이 생성되지 않는다.
7. Mag_Queue()	/ 프로그램이 종료되지 않고 계속해서 반복하게 해주는 함수
8. Render()	/ 생성된 윈도우 창에 Initvb에서 저장한 그림을 출력하는 함수

핵심함수
<윈도우 창 관련 함수>
1. CreateWindow()
2. UpdateWindow()
3. ShowWindow()
<원하는 그림을 출력하기 위함 함수>
1. InitD3D()
2. InitVB()
3. Mag_Queue()
4. Render()
